# 演示脚本草案

## 结论

这个脚本面向比赛现场演示和录制视频，目标不是讲实现细节，而是让评委在 2 到 4 分钟内看懂三个点：场景真实、链路完整、软硬结合有深度。

## 演示目标

- 让评委先理解问题，而不是先看代码
- 明确展示“环境感知 + 动作感知 + 语音交互 + 云端智能”四条链路
- 把项目从普通智能硬件拉到“可落地的康养训练助手”定位

## 建议演示流程

### 1. 开场

- 介绍对象：居家练八段锦、太极的人群
- 说明痛点：动作容易错、环境容易忽略、交互门槛高
- 引出产品：`AuraMate / AI 气韵小师`

### 2. 开机与环境感知

- 佩戴设备
- 屏幕显示启动状态
- 语音播报环境提醒
- 强调 BME690 对空气质量、温湿压的作用

### 3. 动作训练

- 进入训练模式
- 展示 BMI270/BMM350 或扩展节点对姿态的感知
- 强调不是简单计步，而是要做动作质量分析
- 引出 DTW 或相似度评分思路

### 4. 语音与云端

- 用户自然说一句需求，例如“今天肩膀有点僵，适合练什么”
- 展示语音输入到云端模型，再回到本地播报
- 强调端侧负责采集和控制，云端负责复杂推理

### 5. 收尾

- 展示训练得分、建议或摘要
- 强调比赛亮点：软硬结合、嵌入式落地、传统文化场景

## 现场讲解关键词

- `ESP32-C5`
- `Bosch 传感器`
- `I2S 音频链路`
- `LCD + RGB 引导`
- `动作评分`
- `端边云协同`

## 当前仓库与脚本对应关系

- `components/drivers/`
  对应传感器、音频、显示和灯效链路

- `components/services/`
  对应环境、动作、语音、云端和 UI 场景编排

- `docs/task-breakdown.md`
  对应后续把演示做成真实闭环的开发节奏
