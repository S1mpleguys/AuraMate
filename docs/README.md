# 文档索引

## 结论

这个目录已经把比赛项目当前需要的核心文档分成了四类：架构、实施、协作、演示。建议团队成员先从本页进入，按角色定位去看对应文档，避免信息分散。

## 一、架构与目录

- `architecture.md`
  说明软件分层、调用链、状态机和后续任务拆分方向。

- `project-layout.md`
  说明仓库目录职责、模块边界和代码应该放在哪里。

## 二、实施与 bring-up

- `bringup-checklist.md`
  用于硬件调试、驱动接入和系统联调时逐项打勾。

- `hardware-open-items.md`
  记录当前仍未确认的硬件事实，防止大家按不同假设开发。

- `task-breakdown.md`
  说明研发节奏、角色分工和首批落地任务。

## 三、演示与比赛材料

- `demo-script.md`
  用于演示视频、现场讲解和答辩串词的骨架。

- `方案一.md`
  原始比赛思路输入。

- `比赛方案文档.md`
  补充的比赛背景资料。

## 四、建议阅读顺序

1. 新成员先看 `project-layout.md`
2. 做硬件 bring-up 的成员看 `bringup-checklist.md` 和 `hardware-open-items.md`
3. 做应用流程和算法的成员看 `architecture.md` 和 `task-breakdown.md`
4. 做演示和答辩的成员看 `demo-script.md`
